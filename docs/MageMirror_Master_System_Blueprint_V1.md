# 一、系统目标定义（System Goal）

项目名称：

MageMirror（魔镜 AI衣橱系统）

系统目标：

构建平台级 AI衣橱系统，支持大量用户，实现：

- AI衣物识别
- AI自动分类
- AI搭配推荐
- AI试穿能力（未来）
- AI消费入口（长期目标）

系统本质：

AI衣橱 ≠ 工具  
AI衣橱 = 数据入口系统 + AI推理系统 + 用户消费系统

# 六、FastAPI 主服务（核心服务层）

当前系统核心：

FastAPI Server

主要职责：

```
接收上传请求
管理用户数据
触发图片处理
调用AI识别
管理衣橱数据
返回结果
```

主要接口：

```
POST /upload-clothes
GET /get-wardrobe
DELETE /delete-clothes
PUT /update-clothes
```

未来接口：

```
POST /match-outfit
POST /recommend
POST /try-on
POST /import-product
```

---

# 七、上传服务（Upload Service）

职责：

处理图片上传逻辑。

处理流程：

```
接收图片
临时保存
触发压缩
触发处理
```

必须支持：

```
多图片上传
批量处理
统一命名
异常处理
```

建议命名：

```
UUID + 时间戳
```

例如：

```
cloth_20260326_uuid.jpg
```

---

# 八、图片处理服务（Image Processing）

```
背景去除
衣物抠图

居中处理

格式统一
```

```
尺寸：1024x1024
背景：透明
格式：PNG
```

```

边缘增强

# 十八、手机拍照能力（核心）

核心结构：

```html

type="file"
accept="image/*"

multiple>

说明：

```

multiple = 多图选择
```

这是入口关键。
